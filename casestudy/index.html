Case Study for IDM 311

Hunter Hampton

Overview

My 2022 Winter quarter at Drexel I took IDM 311 - UI Design for Immersive Media, a class to introduce us to and build interfaces in Unity.

The goal of this final project was to create multiple user friendly UI and environment interactions in a 3D platform.

Problems

  1. My very first problem I ran into immediately was changing a GameObject's transform property, if it's not in the top level of scene hierarchy, it will likely not move where you expect it to.

    2. The solution was researching why this would be happening on Unity's public forms. I discovered that a nested GameObject position is relative to the parent object, therefore you should use the localPosition property instead of the position property.

  2. Tracking Recast hits using hard coded object names was inefficient, this became problematic once I started re-naming objects, the recast hit would never be detected, failing silently because it didn't produce an error, Unity just thought the object I clicked didn't match the name I entered.

    3. The solution was to not hard code the GameObject names in the hit detection, but to use the the “name” property on the public variables.

  3. The basic Raycasting mouse click test passes through interfaces. This was a problem because one of the main environment interactions is clicking the mountains, which the settings menu is placed in front of, so a single click was both toggling checkboxes and raising/lowering the mountains.

    4. The solution was to limit the Raycast physics check to only listen for hits on a specific Unity layer, a new layer I made only for intractable objects.

Deliverables

    3 UI Controls:

  1. A “Auto Rotate” checkbox toggle to play and pause the sun's rotation.
  2. When the “Auto Rotate” checkbox is disabled, a range slider appears allowing manual adjustment of the sun's position in the sky to affect lighting conditions.
  3. Checkbox toggle to mute/un-mute background music.

    4. 3 GameObject interactions

  4. Clicking on the first blue icon stone will drain or fill the pond
  5. When the “Auto Rotate” checkbox is disabled, a range slider appears allowing manual adjustment of the sun's position in the sky to affect lighting conditions.
  6. Checkbox toggle to mute/un-mute background music.

Outcome & Reflection

    Learned:

  • Use rigid body and mesh/box colliders
  • Write Unity scrips and increase knowledge with using C# class based coding
  • Accept keyboard and mouse input, including the use of Raycasting
  • Manipulate gravity and velocity for movement and jumping
  • Principals of interface design in a game environment
  • Smooth object transforming
  • Make basic materials

If I had more time I would have simply put more intractable features in like a skybox with stars and constellations and solved a bug with Raycasting where if the ray of your pointer passes through multiple clickable GameObjects from different scripts, it will trigger both. And how to organize scripts and game logic better, including how to run scripts that aren't attached to a game object as a component.

Overall I am satisfied with what I was able to learn and accomplish in the given time, having no experience with Unity prior to this project.
